# PCA RFC Template

## 1. Motivation
- Problem statement
- Why existing text is insufficient

## 2. Proposed spec diffs
- Files and sections changed
- Normative language additions/removals

## 3. Security impact
- Threat model changes
- New failure modes and mitigations

## 4. Test updates
- Required additions/modifications to `PCA-TEST-v1`
- New or changed vectors

## 5. Backward compatibility
- Compatible / conditionally compatible / breaking
- Migration guidance

## 6. Version bump rules
- Patch: editorial/clarifications
- Minor: additive compatible semantics
- Major: behavior or schema breaks
