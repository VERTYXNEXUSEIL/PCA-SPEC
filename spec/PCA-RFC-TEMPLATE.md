# PCA RFC Template

## Metadata

- **RFC ID**: `PCA-RFC-XXXX`
- **Title**:
- **Author(s)**:
- **Status**: Draft | Review | Accepted | Rejected
- **Created**:
- **Target release**:

## 1. Summary

Short description of the proposed change.

## 2. Motivation

What problem is being solved and why it matters.

## 3. Scope

Which areas are affected (for example: `constraints_digest`, `effective_time`, Bronze/Silver/Gold profiles, reason codes).

## 4. Proposed specification

- Normative changes (`MUST`, `SHOULD`, `MAY`).
- JSON schema updates (field names stay in English, for example `constraints_version_id`).
- CLI updates (subcommand names stay in English).

## 5. Backward compatibility

Impact on existing implementations and migration path.

## 6. Security considerations

Impact on determinism, digest integrity, TOCTOU handling, OOD handling, and fallback behavior.

## 7. Test plan

Mapping to `PCA-TEST v1` (for example T1-T13) and any additional vectors.

## 8. Operations and rollout

Rollout steps, monitoring, and rollback conditions.

## 9. Open questions

Outstanding questions that block final acceptance.

## 10. Acceptance criteria

Clear and measurable release criteria for accepting the RFC.
