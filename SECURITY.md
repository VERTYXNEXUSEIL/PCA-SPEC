# Security Policy

## Reporting vulnerabilities

Please report potential vulnerabilities privately through repository security reporting features or a maintainer-designated contact placeholder: `security-contact-placeholder`.

Do not publish exploit details before a coordinated fix is available.

## Process

1. Acknowledge report.
2. Reproduce and triage impact.
3. Prepare patch and tests.
4. Coordinate disclosure.

## Scope

- `spec/`, `schemas/`, `test-vectors/`
- `reference/python/` and `reference/cli/`
- `.github/workflows/`
