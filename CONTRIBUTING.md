# Contributing

Thanks for helping evolve PCA/CAE.

## Contribution Types
- Spec clarifications and errata
- New test vectors
- Reference implementation improvements
- Tooling and CI hardening

## Process
1. Open an issue describing motivation and compatibility impact.
2. Submit a PR with clear scope and tests.
3. Preserve normative language consistency in `spec/`.
4. Update `CHANGELOG.md` under **Unreleased**.

## Commit and PR Guidance
- Keep commits atomic.
- Include rationale and verification steps.
- For security-relevant changes, reference threat assumptions.

## DCO / Sign-off (Recommended)
Add `Signed-off-by:` trailers where your organization requires it.
